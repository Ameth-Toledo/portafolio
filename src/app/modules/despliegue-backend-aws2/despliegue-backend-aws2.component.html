Despliegue de BackEnd (Api Rest) en AWS EC2 💪🔥

El despliegue de un backend API REST desarrollado en Java con Javalin en un servidor AWS EC2 permite ofrecer servicios web escalables y eficientes en entornos de producción. Este proceso incluye la configuración del entorno de ejecución de Java, la instalación de dependencias necesarias, la exposición de endpoints REST, la gestión de puertos y variables de entorno, así como la optimización del rendimiento y la seguridad del servidor para garantizar una comunicación estable y confiable con las aplicaciones cliente.

Habilitación de puertos

A continuación, seleccionamos nuestra instancia de backend y accedemos a la sección de Seguridad. Luego, hacemos clic en el enlace de Grupos de seguridad, lo que nos llevará a una nueva ventana donde podremos ver todos los puertos habilitados. Finalmente, damos clic en Editar reglas de entrada para realizar los ajustes necesarios.

Ahora regresamos a Visual Studio Code y abrimos el archivo Main.java para identificar el puerto en el que se ejecuta nuestra API. En mi caso, está corriendo en el puerto 8080. Sabiendo esto, damos clic en Agregar regla y, en el campo Tipo, lo dejamos como TCP personalizado. En el campo Intervalo de puertos, escribimos el número del puerto (en mi caso, 8080) y, en Origen, seleccionamos 0.0.0.0/0.

De tal manera que no debe quedar como se muestra en la imagen. Una vez configurado todo, damos clic en Guardar reglas y listo; aparecerá otra ventana indicando que las reglas del grupo de seguridad se actualizaron correctamente.
